# Інструкція та звіт з модульного тестування (Unit Testing) серверної частини EatPan

Цей документ описує підхід до тестування, інструкції для запуску тестів та звіт про поточне покриття коду.

## 1. Інструкція для запуску тестів

Проєкт використовує фреймворк `pytest` разом із розширеннями `pytest-django`, `pytest-cov`, та `pytest-mock`. 

**Для запуску тестів виконайте наступні кроки в терміналі:**

1. Увійдіть у віртуальне середовище проєкту:
   ```bash
   source .venv/bin/activate
   ```
2. Виконайте запуск тестів із перевіркою покриття (coverage):
   ```bash
   pytest --cov=recipes --cov=eatpan_core --cov-report=term-missing
   ```
3. Щоб згенерувати детальний HTML-звіт з покриттям, виконайте:
   ```bash
   pytest --cov=recipes --cov=eatpan_core --cov-report=html
   ```
   Після цього відкрийте файл `htmlcov/index.html` у вашому браузері.

**Конфігураційні файли тестування:**
- `pytest.ini`: Встановлює змінну оточення `DJANGO_SETTINGS_MODULE = eatpan_core.test_settings` та задає стандартні параметри збору покриття.
- `eatpan_core/test_settings.py`: Конфігурація бази даних in-memory (SQLite) замість PostgreSQL, та dummy-кеш для прискорення тестів і уникнення побічних ефектів.
- `.coveragerc`: Виключає з обрахунку файли міграцій (`migrations/`), скрипти керування (`management/commands/`) та файли ініціалізації (`__init__.py`, `asgi.py`, `wsgi.py`).

## 2. Звіт про тестування та покриття (Test Coverage)

На поточний момент розроблено набір із 22 юніт-тестів, що охоплюють ключову бізнес-логіку застосунку.

**Загальне покриття цільових модулів (recipes, eatpan_core) складає близько 96%**, що є відмінним показником для MVP-версії продукту та відповідає стандартам безпеки розробки (більше цільових 75%).

### Деталізація по компонентах:

1. **Моделі даних (Models)** - `recipes/tests/test_models.py` *(Покриття 100%)*
   - Перевірено створення сутностей `Recipe`, `UserProfile`, `RecipeBook`, `MediaAsset`.
   - Перевірено коректність повернення строкових репрезентацій (`__str__`).
2. **Серіалізатори (Serializers)** - `recipes/tests/test_serializers.py` *(Покриття 100%)*
   - Перевірено серіалізацію вкладених JSON-структур (рецепти, книги рецептів, медіа).
   - Перевірено наявність необхідних полів (`uuid`, `created_at`, тощо).
3. **Механізм Outbox (Синхронізація)** - `recipes/tests/test_sync_outbox.py` *(Покриття 100%)*
   - Перевірено хук `transaction.on_commit` та автоматичне формування подій синхронізації при успішній транзакції.
4. **Контролери (Views & API)** - `recipes/tests/test_views.py` *(Покриття ~90%)*
   - Протестовано CRUD-операції (створення, читання, редагування, видалення) для рецептів та їх груп.
   - Перевірено логіку додавання в улюблене (`toggle_like`).
   - Відпрацьовано відмови у доступі (HTTP 401/403) для неавторизованих запитів.
5. **Автентифікація (Supabase JWT)** - `eatpan_core/tests/test_authentication.py` *(Покриття 100%)*
   - Написані складні тести з використанням моків (Mocking) `jwt.decode` та `PyJWKClient`.
   - Протестовані успішні та неуспішні сценарії (прострочений токен, неправильний алгоритм підпису, невалідний токен) для алгоритмів `HS256` та `ES256`.
6. **Health Check (Моніторинг)** - `eatpan_core/tests/test_health.py` *(Покриття 100%)*
   - Протестовано коректне повернення статусу `200 OK` при наявності з'єднання з базою даних, та `503 Service Unavailable` при його відсутності.

## 3. Рекомендації до подальших тестів (Next Steps)

Хоча модульні тести зараз покривають більшість інфраструктурного коду, у майбутньому можна додати наступні види тестування:

1. **Інтеграційні тести (Integration Tests)**:
   - Розгорнути тестовий контейнер PostgreSQL та NATS JetStream (за допомогою `Testcontainers`) для перевірки реальної взаємодії `Publisher` та `Consumer`.
2. **End-to-End (E2E) тестування**:
   - Перевірка взаємодії між клієнтською (фронтенд) та серверною частинами за допомогою інструментів на кшталт `Cypress` або `Playwright`.
3. **Навантажувальне тестування (Load/Stress Testing)**:
   - Використати `Locust` або `JMeter` для випробування endpoint'ів (наприклад, `/api/v1/recipes/` або `/api/health`) під великим навантаженням.
