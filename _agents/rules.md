# Локальні Інструкції для Бекенда (EatPan Backend)

Ти знаходишся в локальному контексті `EatPan_Back`. Твої завдання тут суворо пов'язані з розвитком, тестуванням та підтримкою логіки бекенда.

Розвиваючи бекенд, дотримуйся наступних правил **High Availability (HA) архітектури**:

1. **Django API (Stateless):** Серверний застосунок Django не повинен зберігати локальний стан (файли, сесії). Всі сесії, токени та rate-limiter повинні зберігатися в In-Memory сховищі (Valkey). Усі файли — в Supabase Storage (або сумісному S3 хранилищі).
2. **Ізоляція контейнерів (Docker):** Valkey та Django API мають працювати у двох окремих незалежних контейнерах в межах одного `docker-compose.yml`. Це дозволяє безпечно запускати, відключати або призупиняти API, не втрачаючи кеш Valkey.
3. **Узгодженість з Архітектурою:** Перед внесенням великих змін обов'язково переглядай `TASKS_AND_STEPS.md`.
4. **Мовні правила:** Коментарі до коду пиши англійською або українською мовами. Markdown-файли — тільки українською.
5. **API Endpoints Synchronization:** Будь-які зміни в API (додавання, видалення, зміна параметрів) повинні обов'язково відображатись у файлі `API_ENDPOINTS.md`. Цей файл має бути завжди синхронізований (скопійований) між проектами: `EatPan_Production`, `EatPan_Fr`, `EatPan_Back`, `EatPan_Supabase`.
